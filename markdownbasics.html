Basics of Markdown

Chapter 1: Introduction

Introduction to Markdown

Markdown is a lightweight markup language created in 2004 by John Gruber with help from Aaron Swartz. It is designed to be easy to read and write, and also easy to convert to HTML and other formats. Markdown is widely used for formatting readme files, for writing messages in online discussion forums, and for creating rich text using a plain text editor.

Why Use Markdown?

  1. Simplicity: Markdown is simple to learn and use. It uses plain text formatting syntax, which is easy to read and write.
  2. Portability: Markdown files are plain text, which makes them portable and compatible across different platforms and devices.
  3. Readability: The syntax of Markdown is designed to be readable as-is, without looking like it’s been marked up with tags or formatting instructions.
  4. Flexibility: Markdown can be easily converted to other formats such as HTML, PDF, and more, making it versatile for various documentation purposes.
Chapter 2: Getting Started

Getting Started with Markdown

Basic Syntax

Markdown syntax is straightforward and consists of a few basic elements:

  • Headings: Use # for headings. More # symbols indicate smaller headings.
  • Emphasis: Use * or _ for italic and ** or __ for bold text.
  • Lists: Use -, +, or * for unordered lists and numbers for ordered lists.
  • Links: Use [text](URL) to create hyperlinks.
  • Images: Use ![alt text](URL) for images.
  • Code: Use backticks (`) for inline code and triple backticks (```) for code blocks.

Here’s an example of Markdown in action:

Heading 1

Heading 2

Heading 3

This is a paragraph with italic and bold text.

  • Item 1
  • Item 2
  • Item 3
  1. First item
  2. Second item
  3. Third item

Link to Google

Inline code: print('Hello, world!')

Code Blocks

In addition to inline code, Markdown allows you to create code blocks, which are useful for displaying longer snippets of code. Code blocks can be created by enclosing the code with triple backticks (```) or by indenting the code with four spaces.

Here's an example of a code block using triple backticks:

def greet(name):
    print(f"Hello, {name}!")

def main():
    name = input("Enter your name: ")
    greet(name)

if __name__ == "__main__":
    main()

The above code block demonstrates a simple Python script that prompts the user for their name and then greets them.

Here's another example of the same code block using indentation:

def greet(name):
    print(f"Hello, {name}!")

def main():
    name = input("Enter your name: ")
    greet(name)

if __name__ == "__main__":
    main()

Both methods will render the code block correctly, preserving the formatting and indentation.

Tools and Editors

Many text editors and integrated development environments (IDEs) support Markdown, often with syntax highlighting and preview features. Some popular choices include:

  • Visual Studio Code: A powerful and extensible code editor with excellent Markdown support.
  • Typora: A minimalistic Markdown editor that provides a seamless live preview.
  • Atom: A hackable text editor with Markdown preview and syntax highlighting.
  • Sublime Text: A sophisticated text editor with Markdown plugins available.

Online tools like Dillinger, StackEdit, and HackMD also offer robust Markdown editing and preview capabilities, making it easy to write and format your documents from anywhere.

Task Lists

Task lists are useful for creating to-do lists or project tracking.

- [ ] Review the draft
- [ ] Submit the final document

Advanced Formatting

Strikethrough

Strikethrough text is supported in some Markdown processors.

Subscript and Superscript

Markdown doesn't natively support subscript and superscript, but you can use simple text markers.

H~2~O for water and E=mc^2^

H2O for water and E=mc^2^
